Installing FLORA-2

First, you need to download and install XSB: either the latest released version or the development version from XSB's CVS repository. Then download FLORA-2 and follow the instructions below.

The best way to instal FLORA-2 is to use a directory outside of the XSB source tree (for instance, ~/FLORA). After unpacking (or checking out from SVN) the FLORA-2 sources will be placed in the subdirectory called flora2. (Under Windows, you need WinZIP in order to unpack FLORA-2 and XSB.)

Configuring FLORA-2 under UNIX

Execute the following commands 

 cd ~/FLORA/flora2
 make clean
 ./makeflora

The last command will work only if an XSB executable is on the system search path specified by the environment variable PATH. If it is not, then additional parameters need to be supplied (here we assume that XSB is installed in the directory ~/XSB): 

 ./makeflora all ~/XSB/bin/xsb

If everything went well, you will be able to run FLORA-2 by typing ~/FLORA/flora2/runflora

Configuring FLORA-2 under Windows

You need Microsoft's nmake, which can be downloaded as part of Visual C++ Express from http://www.microsoft.com/visualstudio/eng/products/visual-studio-express-products#2010-Visual-CPP. Install Visual C++.

Use the following commands to configure FLORA-2 (assuming that XSB is installed and configured, its executable is in D:\XSB\bin\xsb.bat, and FLORA is installed in D:\FLORA\flora2): 

   PATH=C:\Program Files\Microsoft Visual Studio 10.0\VC\BIN;%PATH%
   cd D:\FLORA\flora2
   makeflora  clean
   makeflora D:\XSB\bin\xsb.bat

This assumes that Visual Studio was installed in the folder named C:\Program Files\Microsoft Visual Studio 10.0. If it is installed elsewhere, make the appropriate adjustments.

If you configured a 64-bit version of XSB, then configure FLORA-2 using 

  makeflora D:\XSB\bin\xsb64.bat

Note: the Windows version of makeflora takes only one parameter, while its Unix counterpart takes two.

It is also recommended that you set the environment variable HOME on your Windows system, if it is not already defined. Environment variables are usually set by opening the System folder, which is located inside the Settings folder. Typically, the HOME variable is set to the directory 

   "C:\Documents and Settings\your-user-name"

Once FLORA-2 is installed, you can run it using the command D:\FLORA\flora2\runflora

Configuring FLORA-2 under Cygwin

Cygwin is a free Unix emulator for Windows. The advantage of using XSB and FLORA-2 under Cygwin is that the system will run faster than under native Windows due to special optimizations that XSB has for the GCC compiler. The main disadvantage of Cygwin is that it is a fundamentally 32 bit system, which can use only up to 3GB of main memory. If your uses of FLORA-2 require more, you should bypass this option in favor of a native 64-bit builds of XSB and FLORA-2.

You need a reasonably full installation of Cygwin. In particular, GCC and the related development tools (such as autoconf, make, etc.) must be installed. It is also desirable to have makedepend, which usually comes with the X11 package of Cygwin. If you are planning to access a database, the ODBC DLL must also be installed. XSB must be configured for Cygwin as well.

Other than that, the installation steps for Cygwin are the same as under Unix.
