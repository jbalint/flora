Installing FLORA-2

First, you need to download and install XSB. Then download FLORA-2 and follow the instructions below.

The most flexible way to instal FLORA-2 is to use a directory outside of the XSB directory tree (for instance, ~/FLORA). After unpacking (or checking out from CVS) the FLORA-2 sources will be placed in the subdirectory called flora2. (Under Windows, you need WinZIP in order to unpack FLORA-2 and XSB.)

Configuring FLORA-2 under UNIX

Execute the following commands 

 cd ~/FLORA/flora2
 make clean
 ./makeflora

The last command will work only if an XSB executable is on the system search path specified by the environment variable PATH. If it is not, then additional parameters need to be supplied (here we assume that XSB is installed in the directory ~/XSB): 

 ./makeflora all ~/XSB/bin/xsb

If everything went well, you can run FLORA-2 by typing ~/FLORA/flora2/runflora

Configuring FLORA-2 under Windows

You need Microsoft's nmake, which can be downloaded from http://download.microsoft.com/download/vc15/Patch/1.52/W95/EN-US/Nmake15.exe. The file Nmake15.exe is a self-extracting archive; when it is run, it extracts the program files for nmake.exe into the same directory where you downloaded Nmake15.exe. Let's say this directory is C:\Nmake.

Use the following commands to configure FLORA-2 (assuming that XSB is installed and configured, its executable is in D:\XSB\bin\xsb.bat, and FLORA is installed in D:\FLORA\flora2): 

   PATH=C:\Nmake;%PATH%
   cd D:\FLORA\flora2
   makeflora  clean
   makeflora D:\XSB\bin\xsb.bat

Note: the Windows version of makeflora takes only one parameter, while its Unix counterpart takes two.

It is also recommended that you set the environment variable HOME on your Windows system, if it is not already defined. Environment variables are usually set by opening the System folder, which is located inside the Settings folder. Typically, the HOME variable is set to the directory 

   "C:\Documents and Settings\your-user-name"

Such a setting will prevent certain problems with upgrading FLORA-2 from earlier versions.

You can then run FLORA-2 using the command D:\FLORA\flora2\runflora

Configuring FLORA-2 under Cygwin

Cygwin is a free Unix emulator for Windows. The advantage of using XSB and FLORA-2 under Cygwin is that the system will run faster than under native Windows due to special optimizations that XSB has for the GCC compiler. You need a reasonably full installation of Cygwin. In particular, GCC and the related development tools (such as autoconf, make, etc.) must be installed. It is also desirable to have makedepend, which usually comes with the X11 package of Cygwin. If you are planning to access a database, the ODBC DLL must also be installed. XSB must be configured for Cygwin as well.

Other than that, the installation steps for Cygwin are the same as under Unix.
